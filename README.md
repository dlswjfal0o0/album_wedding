# 우리 가족 앨범 — 설정 및 배포 가이드

가족·지인 5~10명이 사진과 동영상을 용량 걱정 없이 올리고, **누가 올렸는지 표시되면서도 한곳에서 같이 모아볼 수 있는** 공유 앨범입니다. 전체보기 / 사진만 / 동영상만, 그리고 사람별로도 필터링할 수 있어요.

## 1. 전체 구조

| 역할 | 사용 서비스 | 비용 |
|---|---|---|
| 실제 사진/동영상 저장 | Cloudflare R2 | 10GB까지 무료, 다운로드(트래픽) 비용은 평생 무료 |
| 계정 정보 / 파일 목록 저장 | Cloudflare D1 | 무료 |
| 로그인·업로드 등 서버 로직 | Cloudflare Pages Functions | 무료 (하루 10만 요청까지) |
| 화면(웹페이지) | Cloudflare Pages | 무료 |

별도 가입 절차 없이, 관리자(본인)가 가족 계정을 미리 만들어두고 아이디/비밀번호로 로그인하는 방식입니다. R2는 10GB까지 무료이고 그 이후로는 1GB당 한 달에 약 20원 정도라, 가족 단위로 쓰기엔 사실상 거의 무료에 가깝습니다.

이 프로젝트는 별도 빌드 과정이나 터미널 명령어(npm, wrangler 등)가 필요 없습니다. Cloudflare 대시보드 화면에서 클릭만으로 설정할 수 있도록 만들었어요.

## 2. 준비물

- Cloudflare 계정 (https://dash.cloudflare.com 에서 무료 가입)
- GitHub 계정 (이전 프로젝트에서 쓰시던 것 그대로)

## 3. R2 버킷 만들기

1. Cloudflare 대시보드 → 왼쪽 메뉴 **R2** 클릭
2. 처음이면 R2 사용 활성화 화면이 나옵니다. **신용/체크카드 등록을 요구하지만, 10GB 무료 한도 안에서는 요금이 청구되지 않습니다** (어뷰징 방지용 본인 확인 절차예요)
3. **"버킷 만들기"** → 이름은 `family-album-files` 처럼 원하는 이름으로 입력 → 생성

## 4. D1 데이터베이스 만들기

1. Cloudflare 대시보드 → 왼쪽 메뉴 **Workers & Pages** → **D1** 탭
2. **"데이터베이스 만들기"** → 이름은 `family-album-db` 로 입력 → 생성
3. 만들어진 데이터베이스로 들어가서 **"콘솔(Console)"** 탭을 클릭
4. 이 프로젝트 안의 `schema.sql` 파일 내용을 전부 복사해서 콘솔에 붙여넣고 실행 — `users`, `files` 두 테이블이 만들어집니다

## 5. GitHub에 올리기

이전에 하셨던 방식과 동일하게, 이 폴더 전체를 새 GitHub 저장소(예: `family-album`)에 올려주세요.

## 6. Cloudflare Pages에 연결하기

1. Cloudflare 대시보드 → **Workers & Pages** → **"애플리케이션 만들기"** → **Pages** 탭 → **"Git에 연결"**
2. 방금 올린 GitHub 저장소 선택
3. 빌드 설정:
   - **프레임워크 프리셋**: None (없음)
   - **빌드 명령어**: 비워두기
   - **빌드 출력 디렉터리**: `/`
4. **"저장 및 배포"** 클릭

## 7. R2 / D1 연결 + 비밀 값 등록 (가장 중요한 단계)

배포가 끝나도 아직 데이터베이스와 저장소가 연결되어 있지 않습니다. 아래를 꼭 해주세요.

1. 방금 만든 Pages 프로젝트 → **Settings(설정)** → **Functions** 탭
2. **R2 bucket bindings** → 추가 → 변수 이름은 정확히 `BUCKET` (대문자) 입력 → 3번에서 만든 버킷 선택
3. **D1 database bindings** → 추가 → 변수 이름은 정확히 `DB` (대문자) 입력 → 4번에서 만든 데이터베이스 선택
4. 같은 Settings 안의 **환경 변수(Environment variables)** 에서 다음 두 개를 추가 (둘 다 "암호화" 옵션으로):
   - `JWT_SECRET` — 아무 긴 임의의 문자열 (예: 32자 이상 랜덤 문자/숫자). 로그인 토큰 서명에 쓰이며 외부에 알려지면 안 됩니다.
   - `ADMIN_SETUP_KEY` — 계정 생성 페이지(admin.html) 보호용 비밀번호. 가족에게 알려주지 않을 본인만의 키입니다.
5. 변경 후 **반드시 다시 배포(Retry deployment 또는 새 커밋)** 해야 적용됩니다

## 8. 가족 계정 만들기

1. 배포된 주소 뒤에 `/admin.html`을 붙여서 접속 (예: `https://family-album.pages.dev/admin.html`)
2. 7-4에서 정한 `ADMIN_SETUP_KEY` 입력
3. 아이디 / 화면에 보일 이름 / 비밀번호(4자 이상) 입력 후 계정 생성 — 가족 인원수만큼 반복
4. 만든 아이디/비밀번호는 가족들에게 직접 알려주세요 (메시지 등으로). **이 admin.html 주소는 가족에게 공유하지 마세요** — 새 계정을 함부로 만들 수 있는 화면이라 본인만 알고 있어야 합니다

## 9. 사용해보기

1. 배포된 Pages 주소(메인 화면, admin.html 아님)로 접속해서 만든 계정으로 로그인
2. 업로드 버튼으로 사진/동영상 선택 — 큰 영상도 자동으로 나눠서 올라갑니다 (용량 제한 거의 없음)
3. 상단 탭에서 **전체 / 사진 / 동영상**, 그 아래 사람별 칩으로 **누가 올렸는지** 필터링
4. 사진/영상을 누르면 크게 보기, 다운로드 가능. 본인이 올린 파일만 삭제 버튼이 보입니다

## 10. 알아두면 좋은 것들

- 다운로드(트래픽) 비용은 R2가 영구적으로 무료라서, 가족들이 사진을 자주 봐도 추가 비용이 없습니다.
- 10GB 무료 한도를 넘으면 1GB당 한 달에 약 20원 정도 청구됩니다 (100GB를 써도 한 달 2천원 수준).
- 영상은 목록에서는 미리보기 아이콘만 보이고, 눌렀을 때만 실제로 재생되어 데이터를 아낍니다.

## 11. 문제가 생기면

- 사진/영상이 안 보이거나 업로드가 안 되면 → Settings → Functions에서 바인딩 이름이 정확히 `BUCKET`, `DB`인지 확인 (대소문자까지 정확해야 합니다)
- 바인딩이나 환경 변수를 바꾼 뒤에는 꼭 재배포해야 적용됩니다
- 계정이 잘 만들어졌는지 확인하려면 D1 콘솔에서 `SELECT * FROM users;` 실행

## 12. 나중에 추가하면 좋은 기능

- 동영상 썸네일 자동 생성 (현재는 재생 아이콘 placeholder)
- 여러 장 한꺼번에 zip으로 다운로드
- 모바일 "홈 화면에 추가" PWA 설정

필요하시면 언제든 이어서 만들어드릴 수 있어요.
